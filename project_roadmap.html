Hoja de Ruta Detallada: Carretilla Elevadora Autónoma

Este documento define la estrategia técnica y las fases de desarrollo para el sistema de carretilla elevadora autónoma. El objetivo es crear un robot industrial capaz de operar de forma segura y eficiente en entornos de almacén utilizando ROS 2 Jazzy.

1. Arquitectura del Sistema

El sistema se basa en una arquitectura modular dividida en Simulación, Navegación, Control y Lógica de Decisión.

Componentes Clave

  • Simulación:
    • Actual: Mvsim (rápido, ligero) para validar lógica de navegación.
    • Objetivo: Gazebo (físicas realistas) para validar manipulación de carga.
  • Navegación (Nav2 Route Server):
    A diferencia de los robots de servicio que navegan libremente, una carretilla industrial debe seguir "carriles" virtuales. Usaremos Route Server para definir un grafo de rutas predecible.
  • Percepción:
    Uso de cámaras y LiDAR para localización (AMCL) y detección de pallets (ArUco / Deep Learning).

2. Fases de Implementación

Fase 1: Cimientos y Navegación (En Progreso)

Objetivo: Lograr que el robot se mueva de forma autónoma entre puntos A y B siguiendo las reglas del almacén.

  • [Hecho] Configuración del entorno Mvsim con mapa de almacén.
  • [Hecho] Integración del robot (Turtlebot3 como base móvil).
  • [Pendiente] Configuración de Nav2 Route Server:
    • Creación del Grafo de Rutas (nodos en pasillos, zonas de carga).
    • Configuración de los controladores de seguimiento de ruta.

Fase 2: Percepción y Aproximación Fina

Objetivo: Permitir que el robot detecte pallets y se alinee con precisión milimétrica para la inserción de las palas.

  • Integración de cámara simulada en el robot.
  • Desarrollo de nodo de detección de marcadores ArUco.
  • Implementación de un controlador de Servoing Visual:
    El robot deja de usar el mapa global y se mueve basándose únicamente en la posición relativa del marcador.

Fase 3: Lógica de Negocio (Máquina de Estados)

Objetivo: Orquestar las operaciones complejas del robot.

  • Implementación de Behavior Tree (BT.CPP) o Máquina de Estados.
  • Definición de estados:
    • IDLE: Esperando misión.
    • NAVIGATING: Moviéndose por el grafo.
    • APPROACHING: Alineándose con el pallet.
    • LIFTING/LOWERING: Operación de carga.
    • ERROR: Fallo de localización o bloqueo.

Fase 4: Manipulación Física

Objetivo: Simular la física real de levantar una carga.

  • Migración a Gazebo.
  • Modelado de la carretilla con articulación prismática (eje Z).
  • Simulación de dinámicas de carga (centro de gravedad, fricción).

3. Futuras Iteraciones (Roadmap a Largo Plazo)

Iteración 5: Flota Multi-Robot

Coordinación de múltiples carretillas en el mismo almacén.

  • Gestión de tráfico centralizada (evitar bloqueos en pasillos estrechos).
  • Asignación de tareas optimizada (Fleet Manager).

Iteración 6: Integración en la Nube

Conexión con sistemas de gestión de almacenes (WMS).

  • Dashboard web para monitorización en tiempo real.
  • API REST para recibir pedidos de movimiento de stock.

Iteración 7: Hardware Real

Despliegue del software en una carretilla física o prototipo.

  • Integración de drivers de motores reales (CAN bus / Modbus).
  • Seguridad certificada (PL-d/SIL-2) con LiDAR de seguridad.
